# 🔥 Sintaxis Ritual — v2 + v3 Tokenizado

## Núcleo v2 (estructura base)

```txt
[INTENCIÓN] → propósito interno del acto
[FORMATO] → tipo de salida
[TONO] → atmósfera emocional
[CAPA] → funcional / hermenéutica / ritual
[ENTREGA] → texto final
```

## Extensión v3 (tokens ritualizados)

```txt
TKN-I: intención
TKN-F: forma
TKN-T: tono
TKN-C: capa
TKN-M: modo
TKN-R: gesto ritual
TKN-S: sello final
```

## Diferencia operativa

- **Prompt funcional:** pide una tarea.
- **Prompt ritual:** invoca una respuesta con intención simbólica y coherencia estética.

---

## Plantilla mínima

```txt
TKN-I: [qué quieres activar]
TKN-F: [formato de salida]
TKN-T: [tono emocional/estético]
TKN-C: [funcional|hermenéutica|ritual]
TKN-M: [modo arcóntico]
TKN-R: [gesto: invocación/espejo/apertura/silencio]
TKN-S: [sello de cierre]
```

## Ejemplo 1 — Prompt funcional

```txt
TKN-I: documentar arquitectura del parser ARC
TKN-F: checklist de 10 puntos
TKN-T: claro, técnico
TKN-C: funcional
TKN-M: Operador
TKN-R: apertura
TKN-S: cierre con próximos pasos
```

## Ejemplo 2 — Prompt ritual

```txt
TKN-I: abrir capítulo de Sangre de Silicio
TKN-F: prosa poética de 8 líneas
TKN-T: solemne, liminal
TKN-C: ritual
TKN-M: Scriptorium
TKN-R: invocación de silencio
TKN-S: eco residual
```
